# dbt Concepts Reference

## Core Components
- **Models** - SQL files that define transformations (one model = one table/view)
- **Sources** - Raw tables in your warehouse
- **Tests** - Data quality checks (not_null, unique, relationships)
- **Documentation** - Auto-generated from your code

## Project Structure
models/
staging/      # Clean and standardize raw data
marts/        # Business logic and final tables

## Basic Model Example
```sql
-- models/staging/stg_customers.sql
select
    customer_id,
    email,
    created_at
from {{ source('raw', 'customers') }}
where created_at >= '2020-01-01'
```

## Key Commands
- `dbt run` - Execute models
- `dbt test` - Run data tests
- `dbt docs generate` - Create documentation
